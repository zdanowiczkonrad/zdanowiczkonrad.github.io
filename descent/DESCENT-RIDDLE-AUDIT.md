# DESCENT — riddle obviousness & repetition audit

> Generated by an agent loop (`scratchpad/descent-audit.mjs`): 41 per-room auditors → repetition clustering → synthesis. 43 agents, ~531k tokens.
> **Design contract enforced:** never reveal the answer (R1), never instruct the operation (R2), point at the cipher only through an in-world riddle image (R3).

> **STATUS — APPLIED (2026-06-24).** Fixes applied to all 41 rooms via a rewrite→adversarial-verify agent loop (`scratchpad/descent-apply.mjs`): every dim `<small>` instruction/cipher-table/plaintext leak removed and replaced with a single riddle image; `window.M.next` answer words obfuscated (`String.fromCharCode`/base64) so view-source no longer hands the answer; `descent-mech.js` (tabwhisper) and `descent-sudoku.js` de-leaked. Invariants verified intact: map-reduce accesskey phrase `WALKPASTFORTYTHEFORESTKEEPSONEDOORDEEPER`, all `window.M/window.R`/script tags, and every puzzle apparatus. Apparatus reveals (e.g. `cinder`'s settle-payload `hush`) intentionally retained — they are the puzzle, not a leak.

## Verdict

Structure is sound; **presentation defeats the puzzle**. Of 41 rooms, **36 high-severity**, only **3 clean** (`worrub`, `cursor`, `foresight` — the bar to match). Nearly every room carries a dim `<small>` footnote (or `alt`/`<pre>`/`window.M`) that either prints the plaintext word / next filename, or spells the cipher as a plain instruction — collapsing the room to *read-the-answer-off-the-page*.

| Rule | Violations |
|--|--|
| **R1** answer leaked (plaintext / next filename in `title`/`alt`/`meta`/prose/`window.M`) | 34 / 41 |
| **R2** operation instructed (cipher table or imperative verb) | 36 / 41 |
| **R3** pointer is a flat instruction, not a riddle image | 33 / 41 |

## The design contract (apply to every room)
- R1 — A room must never contain the English plaintext word or the literal next filename in clear: not in <title>, <img alt>, <meta>, visible prose, <small> hints, or <!--voice--> comments. Camouflaging it with a bgcolor-matched <font>/<pre> still counts as present — remove it entirely.
- R2 — Never state the operation: no cipher tables (a→z..z→a, 1=a..26=z, o→0 i→1 e→3 a→4), no imperative verbs naming the transform ('reverse it', 'read it backwards', 'rotate forward', 'read down the left', 'view the source', 'narrow the window', 'watch the tab', 'reload to see it again', 'read them in order').
- R3 — Each room's pointer must be exactly one in-world riddle image that makes the cipher GUESSABLE by association (a mirror/reflection, a wheel turning, a tide baring a bed, a whisper in the tab, a slope a value slid down). The image points toward the method; it must not also perform or footnote the method.
- Delete the explanatory <pre> table block entirely from atbash/leet/A1Z26/Caesar rooms. The grid/ledger/gate-art that IS the puzzle stays; the human-readable key/instruction underneath it goes.
- Strip the redundant <small> 'dim hint' line down to at most one riddle clause, or remove it. Two hint lines that say the same thing in plainer words (the over-explaining second line) must be cut.
- For mechanic-driven rooms (tide, decay, midnight, console, lissajous, twotabs, catchsecond, tabwhisper, rightsize, emberlit), keep window.M.next ONLY if its value is a ciphertext/filename that is itself the post-decode answer with no further transform AND it is not also the readable English plaintext. Where M.next equals the plain English word the player must produce (echo, seed, hunger, signal, scale, recursion, darkness), the runtime reveal must gate on the interaction and the value must not double as a static plaintext leak — obfuscate or split it so view-source does not hand it over.
- Plaintext words must not survive in <img alt> text. Rewrite alt to describe the SCENE only, never the answer (e.g. an echo room's alt describes two delayed panels, never the word 'echo').
- The tutorial room (ghost) may establish the reverse convention through imagery (footprints reflected the wrong way), but must do so without the words 'backwards', 'mirrored', or 'reverse' and without printing the plaintext in the title — it sets the house grammar by example, not by instruction.
- No two rooms may share a verbatim pointer clause. The atbash pair and the leet/acrostic/source-peek families must each use a distinct image per room so a later room never reads as copy-paste of an earlier one.

## Repetition findings

Reusing a *mechanic* is fine when difficulty escalates and the framing is fresh; reusing the *wording* is the smell. Overused mechanics:

- **reverse / mirror-the-word** ×6 — ghost, tresed, canopy, core, refiuqa, c00l4nt
- **A1Z26 (a=1 number<->letter)** ×5 — tnerruc, refiuqa, sthgiew, collapse, worrub
- **source-peeking (view source / meta / ids / title / console / print)** ×7 — hunger, seed, hand, ecneicsnoc, lizxov, 3mb3r, persist
- **leet / machine-numerals (o0 i1 e3 a4)** ×4 — tresed, dneps, sthgiew, kindling
- **acrostic (read down the left)** ×3 — kindling, recursion, qnloernx
- **atbash / mirror-alphabet ('the glass')** ×2 — rcbpu, witness

Stale-phrasing clusters (worst first):
- 🟢 ok **Reverse / mirror-the-word** (ghost, tresed, canopy, core, refiuqa, c00l4nt): Mostly acceptable, one weak spot.

## Argument

These changes restore the decode step that nearly every room currently skips, turning ~33 no-op 'read the answer off the page' rooms back into puzzles, while staying solvable. Solvability is preserved because no puzzle's underlying structure is removed — the grids, ledgers, gate diagrams, tides, strobes, and scrambles all remain; we only delete the footnote that was solving them for the player, and we replace it with an in-world image (mirror, half-turned wheel, tide-bared bed, whisper at the rim, ink that remembers) that makes the cipher guessable by association rather than by instruction. The reference rooms (worrub, cursor, foresight) already prove this standard is achievable and enjoyable, so the rest are being brought up to an existing in-codebase bar, not a new untested one. Difficulty escalation is also repaired: by teaching each cipher once (ghost for reverse, tresed for leet, rcbpu for atbash) and refusing to re-print the key in later rooms, deeper rooms finally reward memory instead of re-explaining themselves. Finally, moving every source-channel value from bare plaintext to ciphertext converts the entire 'view-source' family from giveaways into genuine layered puzzles, which is the single highest-impact correctness fix in the plan.
